# Рефакторинг проекта Pawn Wars

## Выполненные изменения

### 1. Модульная структура

Создана новая модульная структура для движка ИИ:

```
engine/
├── factors/              # Факторы оценки позиции
│   ├── FactorRegistry.js      # Реестр факторов
│   ├── BaseFactor.js          # Базовый класс
│   ├── LegacyAdapter.js      # Адаптер для старого формата
│   ├── utils.js              # Утилиты
│   ├── PawnCount.js          # Пример фактора
│   ├── PawnAdvancement.js    # Пример фактора
│   └── ExampleNewFactor.js   # Шаблон для новых факторов
├── search.js             # Алгоритм поиска (Minimax)
├── evaluator.js          # Оценка позиции
├── config.js             # Конфигурации ИИ
└── index.js              # Главный модуль
```

### 2. Система регистрации факторов

Создана система `FactorRegistry` для регистрации и управления факторами оценки. Это позволяет:
- Легко добавлять новые факторы
- Автоматически загружать факторы
- Использовать факторы как в браузере, так и в Node.js

### 3. Обратная совместимость

- Старый файл `factors.js` продолжает работать
- `ai.js` обновлен для использования новой системы, но сохраняет старый API
- Все существующие скрипты (`run_ai.js`, `gridsearch.js`) продолжают работать

### 4. Расширяемость

Теперь легко добавлять новые факторы оценки:

1. Создайте файл в `engine/factors/`
2. Реализуйте класс фактора с методом `evaluate()`
3. Зарегистрируйте фактор в `FactorRegistry`
4. Используйте в конфигурации ИИ

См. `engine/README.md` для подробных инструкций.

## Использование

### В браузере

Все модули загружаются автоматически через script теги в `index.html`.

### В Node.js

```javascript
// Загружаем движок
const { findBestMove, makeAiMove } = require('./engine');

// Используем как обычно
const result = findBestMove(7);
```

## Миграция существующих факторов

Существующие факторы из `factors.js` автоматически загружаются через `LegacyAdapter`. Для полной миграции:

1. Скопируйте функцию оценки из `factors.js`
2. Создайте новый файл в `engine/factors/`
3. Преобразуйте в класс фактора (см. примеры)
4. Зарегистрируйте фактор

Старые факторы продолжат работать до полной миграции.

## Преимущества новой структуры

1. **Модульность**: Каждый фактор в отдельном файле
2. **Расширяемость**: Легко добавлять новые факторы
3. **Тестируемость**: Каждый модуль можно тестировать отдельно
4. **Читаемость**: Код лучше организован
5. **Совместимость**: Работает и в браузере, и в Node.js
